Angular 8.0 有什么新功能?

本文字数： 6.8k | 阅读时长 ≈ 6 分钟

Angular 8.0.0 正式发布了!

首先，请允许我做一次个人公告：Angular 的核心团队正在努力吸收更多来自于社区的开发人员，我现在正式以合作者的身份成为了 Angular 团队的一员。在 Angular 8.0 中，我贡献了相当多的代码 😊。

这个版本主要是关于 Ivy 可能性的一些尝试，同时也包含了一些新特性和重大变化。因此，Angular 团队编写了一堆 schematics，希望这些工具可以帮你把繁重的工作简化得尽可能的简单。

TypeScript 3.4

Angular 8.0 已经支持 TypeScript 3.4，甚至是必须使用它，所以你需要升级到 TypeScript 3.4。
你可以在微软官方博客上查看 TypeScript 3.3 和 TypeScript 3.4 之间有些什么新特性。

Ivy

Ivy 显然是这个版本的重要的组成部分，为此团队在上个月花费了大量的精力。关于 Ivy 有很多内容可以介绍，因此我专门写了一篇有关于 Ivy 的文章。

很重要的一点: Ivy 是 Angular 新启用的编译器(compiler)/运行时(runtime)。它将在未来给 Angular 带来非常酷的特性，但目前的重点是如何让现有的应用程序平滑升级。

Angular 8.0 是第一个正式提供可选项切换到 Ivy 的版本。虽然现在这样做并没有一些实质性的好处，但是您可以借此机会尝试一下，看看您的应用程序中是否具备平滑升级到 Ivy 的可能性。因为在未来某个时间点，有可能在 v9 中，Ivy 将会成为默认提供的。因此，Angular 团队希望社区能够预料到这种转变并提供反馈，并且我们在 v9 之前可以关注到所有剩余的问题。

我们在几个应用程序上尝试了它并且已经发现了一些问题，所以我们强烈建议不要盲目地在生产环境中使用它😄。

当然，如果您想感受一下冒险精神，可以在 angularCompilerOptions 中添加 "enableIvy"：true，然后重新启动您的应用：它现在就已经使用 Ivy 了！ 查看我们的文章和官方指南以获取更多信息。

Bazel support

至于 Ivy，我们专门写了一篇关于如何使用新的 Bazel 构建 Angular 应用程序的文章🛠。

Forms

markAllAsTouched

除了现有的markAsDirty，markAsTouched，markAsPending等等之外，AbstractControl类现在提供了一种新的方法:markAllAsTouched。由于AbstractControl是FormGroup，FormControl，FormArray的父类，因此该方法适用于所有响应式表单实体(Reactive Form Entities)。

和markAsTouched一样，这个新方法将控件标记为touched，并同时对它所包含的所有子孙级控件生效。

FormArray.clear

FormArray类现在提供了clear方法，以快速删除它包含的所有控件。在以前，您必须循环所有控件以逐个删除它们。

Router

使用 import() 语法的延迟加载

Location

为了帮助大家从 AngularJS 升级，Angular 中的位置服务(Location Services)中增加了不少东西。

PlatformLocation现在提供了对hostname、port和protocol的访问，而新的getState()方法则允许获取history.state。 MockPlatformLocation可用于简化测试。如果你使用ngUpgrade，这一切都非常好用，否则你也可能不需要它。

Service worker

Registration strategy

service worker 的注册方式有了一个新选项，允许指定何时进行注册。此前，service worker 始终在应用程序运行稳定后注册，以避免减缓应用程序的启动。但是，如果您在应用程序的引导程序上重复的启动异步任务（比如轮询过程），则应用程序永远不会进入稳定状态，因为只有在没有挂起的任务时，Angular会认为应用程序是稳定的。所以这会导致 service worker 永远无法注册，你必须手动处理它。现在，您可以使用新的 registrationStrategy 选项，让 Angular 来处理这个问题。有几个可选的值：

  • registerWhenStable，默认值, 就像上面说的那样。
  • registerImmediately，不等待应用程序稳定并立即注册 service worker
  • registerDelay:$TIMEOUT 等待 $TIMEOUT 毫秒之后立即注册
  • 一个返回 Observable 的函数，用于定义自定义策略。然后，service worker 将在 Observable 发出其第一个值时进行注册。

Multiple apps on sub-domains

之前，在同一个域的不同子路径上运行多个使用 @angular/service-worker 的应用程序是完全不可能的，因为每个 Service Worker 都会覆盖其他的缓存， Emmmm…，现在这个问题已经修复了！

值得注意的重大更新

有一些变化需要你做一些准备工作。一些变化是由 Ivy 驱动的，并且是为了让我们的应用程序做好准备。但很酷的消息是，Angular 团队已经编写了 schematics，以使我们更轻松的应对这些变化。

只需运行 ng update @angular/core，schematics 将更新您的代码。都有哪些变化呢，我们来看看吧！

DOCUMENT

DOCUMENT 从 @angular/platform-browser 移动到 @angular/common。您可以在应用程序中手动更改它，时，提供的 schematic 也可以为您自动处理。

Deprecated webworker package

@angular/platform-webworker 包启用了在 Web Worker 中运行 Angular 应用程序的特性。但这已经被证实比预期的结果要糟糕的多（用于构建应用程序，SEO……），而且性能也不是很好，因此该软件包已被弃用，在将来将被移除。

Deprecated HTTP package removed

@angular/http 在8.0中已经被移除，在18个月之前的4.3中被@angular/common/http替换，并在5.0中被正式弃用。您可能已经迁移到@ angular/common/http了，但如果没有，现在您可能必须要迁移了，提供的 schematic 会从 package.json 中删除依赖项。

您还可以在官方弃用指南中找到所有已弃用的API。

这就是Angular 8.0的全部内容！ 您也可以查看我们其他一些关于Ivy，CLI 8.0版本或新 Bazel 支持的相关文章。
